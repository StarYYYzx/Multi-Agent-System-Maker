一、项目总览（已确认全部MVP需求）
 
1. 项目名称
 
零代码多Agent拖拽式工作流设计平台（低代码Agent开发系统MVP）
 
2. 核心目标
 
面向无代码业务人员，通过可视化画布拖拽搭建多智能体协作链路；用户仅填写任务描述，系统自动生成标准化Prompt，完成多Agent串行、分支、并行协作。
 
3. 约束边界（MVP严格限定）
 
1. 能力：仅文本对话Agent，预留工具调用、知识库、API扩展接口
2. 模式：纯零代码，无脚本输入
3. 存储：浏览器本地LocalStorage存储蓝图，运行日志本地持久化
4. 执行：单轮大模型调用，无节点内部多轮反思，预留拓展开关
5. 异常：调用失败立即终止流程，全量记录每一步运行数据
6. 交互：左侧节点库拖拽+手动连线；自带4套分类Prompt模板，支持保存自定义工作流蓝图；配备撤销操作（50步栈），无重做，无版本管理
7. 执行时限：整体流程最长运行2分钟，无自动重试机制；单节点超时通过 AbortController 中断 HTTP 请求
8. API Key 管理：支持页面内可视化配置（/settings），Fernet 加密存入服务端临时文件，重启自动丢失
 
 
 
二、功能需求详细规格（FR）
 
模块1：可视化画布编辑器
 
1.1 节点组件库
 
基础节点（必选）
 
1. 开始节点：全局唯一入口，接收用户初始文本输入，固定输出JSON结构化数据。
2. Agent对话节点：核心业务节点
- 用户输入：自然语言任务描述
- 内部逻辑：模板池匹配+AI自动生成Prompt
1. 优先从内置Prompt模板库匹配对应场景模板
2. 匹配成功：将任务自动填充到模板，生成角色设定、任务指令、固定输入输出格式
3. 无匹配模板：调用大模型自动生成完整结构化Prompt，强制约束返回格式
- 输出：结构化JSON字符串，保证下游节点可预处理
3. 结束节点：流程出口，汇总最终结果并对外输出。
 
逻辑控制节点（MVP内置）
 
1. 条件分支节点
- 配置方式：用户用自然语言描述分支判断规则（方案A）
- 运行机制：大模型读取上游输入+判断语句，输出指定分支编号
- 数据传递：向下传递标准化结构化报文
2. 并行分支节点：将同一份输入同时分发至多条子链路
3. 数据汇总节点
- 核心规则：接收多条并行链路的结构化输出，由AI按照预定义格式整理合并，统一向下游传递结构化数据，杜绝纯自由文本拼接。
 
1.2 画布交互规则
 
1. 操作方式：左侧面板拖拽节点至画布，鼠标拖拽连接线构建有向链路（DAG）
2. 基础操作：选中、移动、删除、撤销（仅撤销最近操作，无重做）
3. 链路约束：禁止循环闭环，编辑器自动校验拓扑结构，检测到环路直接给出提示。
4. 主Agent智能生成：点击工具栏按钮，系统读取完整DAG结构（节点类型+连线关系），调用大模型分析流程意图，自动为每个Agent节点生成贴合上下游关系的taskDescription。

1.3 蓝图资产管理

1. 内置资源：预装4套分类Prompt模板（文案创作/客户答疑/信息总结/内容审核），关键词匹配自动填充
2. 用户资产：支持将画布工作流保存为本地蓝图JSON文件，存储在浏览器本地存储
3. 导入：可读取本地保存的蓝图，一键恢复画布布局与全部节点配置。
 
模块2：工作流运行引擎
 
2.1 两种运行入口
 
1. 画布内调试：直接在编辑器点击【运行】，弹窗录入初始用户输入
2. 流程执行页：独立页面运行已保存的蓝图
 
2.2 运行可视化
 
1. 执行链路实时高亮，当前活跃节点标红
2. 分步展示每一个节点：入参、调用Prompt、大模型原始返回、处理后结构化输出、节点耗时
 
2.3 数据格式约定（保障代码预处理）
 
所有节点之间传递数据统一为固定Schema的JSON：
 
json
  
{
  "sourceNodeId": "节点唯一ID",
  "rawContent": "原始文本",
  "structuredData": {},
  "timestamp": 1786500000
}
 
 
2.4 日志系统
 
1. 每次执行完整日志持久化到浏览器本地
2. 日志内容包含：整体起止时间、每个节点入参、出参、执行时长、模型调用报文
3. 调用异常（超时、接口报错）：立刻终止整个工作流，在日志中标注失败节点与错误信息。
 
2.5 执行约束
 
- 总执行上限：120秒，超时自动强制终止
- 无自动重试逻辑。
 
模块3：Prompt自动生成引擎（双模式结合）
 
1. 内置分类模板池：文案创作、客户答疑、信息总结、内容审核四类标准化模板
2. 路由逻辑：
- 第一步：使用关键词匹配判断用户任务属于哪一类场景（MVP阶段用关键词表+正则匹配，后续可升级为LLM分类）
- 第二步：命中模板 → 自动填充任务，强制设定IO格式
- 第三步：未命中任何模板 → 调用生成模型，自动编写角色人设+任务指令，并强制要求输出结构化JSON
3. 架构预留：模型配置项抽离为配置文件，后续可快速切换多家大模型接口。
 
模块4：扩展预留接口（非业务功能，架构层预留）
 
1. 节点插件插槽：预留空白自定义节点入口，后续可扩展API调用、工具执行、知识库检索节点
2. 功能开关配置：Agent节点预留「单轮调用/多轮反思」布尔开关，后续迭代开启多轮推理
3. 并行分支数量配置：后端配置项控制最大并行数，无需修改业务代码。
 
 
 
三、非功能需求（NFR）
 
1. 存储：所有蓝图、运行日志全部保存在浏览器LocalStorage，不上云，无需账号登录
2. 拓扑校验：前端实时校验DAG，阻止循环流程生成
3. 兼容性：支持Chrome、Edge主流桌面浏览器
4. 数据规范：全链路报文结构化，便于后续后端服务对接、数据清洗与预处理
5. 可扩展性：前后端分层解耦，工作流引擎与画布UI完全分离。
 
 
 
四、系统架构设计（软件工程分层设计）
 
整体架构（前后端分离，MVP前端可独立运行，后端仅提供大模型API转发服务）
 
四层分层架构
 
1. 表现层（前端）
- 可视化画布：基于开源流程图组件（GoJS / X6）开发拖拽编辑器
- 蓝图管理页面、运行调试面板、日志查看面板
- 本地存储模块：序列化画布数据，写入LocalStorage
2. 业务编排层（前端/轻量后端）：DAG工作流引擎
- 拓扑解析器：解析画布节点与连线，生成执行拓扑顺序
- 调度器：串行执行、并行分发、分支路由调度
- 格式处理器：统一入参、出参JSON格式化
3. 智能服务层（后端接口服务）
- Prompt生成服务（模板匹配+自动生成双逻辑）
- 主Agent工作流分析服务（解读DAG结构，智能生成各节点任务描述）
- 分支条件判断服务（自然语言条件识别）
- 内容汇总整理服务
- 大模型网关：统一封装多家LLM调用接口，方便后续切换模型
4. 基础设施层
- 本地浏览器存储（MVP）
- LLM厂商API（OpenAI/国内大模型接口）
 
核心执行流程
 
1. 用户在画布拖拽节点，配置每个Agent节点的任务描述，保存蓝图JSON
2. 用户输入初始问题，启动工作流
3. 引擎解析DAG拓扑，按顺序执行节点：
1. Agent节点 → 请求Prompt服务生成结构化指令
2. 调用大模型，接收返回数据
3. 格式化输出为标准JSON报文，向下一节点传递
4. 条件分支节点调用判断服务，自动选择运行分支
5. 并行链路执行完毕后，汇总节点调用AI合并多路结果
4. 全程记录节点耗时与IO数据，出现异常立刻终止并写入日志
5. 最终输出汇总结果。
 
 
 
五、核心数据模型（软件工程实体设计）
 
实体1：工作流蓝图 WorkflowBlueprint（JSON序列化存储）
 
typescript
  
{
  id: string; // 蓝图唯一标识
  name: string; // 蓝图名称
  nodes: Node[]; // 全部节点集合
  edges: Edge[]; // 节点连接线
  createTime: number;
}
 
 
节点实体 Node
 
typescript
  
{
  nodeId: string;
  nodeType: "start" | "agent" | "condition" | "parallel" | "merge" | "end";
  config: {
    taskDescription?: string; // Agent节点：用户填写的任务
    branchRule?: string; // 分支节点：自然语言判断条件
    templateId?: string; // 匹配到的Prompt模板ID
  },
  position: {x: number, y: number} // 画布坐标
}
 
 
连线实体 Edge

typescript
  
{
  edgeId: string;
  sourceNodeId: string;
  targetNodeId: string;
  label?: string;         // 分支标识，如 "pass"、"reject"，用于条件分支节点出口标记
  branchIndex?: number;   // 分支数字索引 0、1、2...，与 label 二选一使用
}
 
 
实体2：单次执行日志 ExecutionLog（含节点级记录）

typescript
  
interface ExecutionLog {
  logId: string;
  blueprintId: string;
  startTime: number;          // 整体开始时间戳
  endTime: number;            // 整体结束时间戳
  status: 'success' | 'failed' | 'timeout';
  errorMessage?: string;      // 整体失败原因
  nodeRecords: NodeRecord[];  // 每个节点的运行记录
}

interface NodeRecord {
  nodeId: string;
  nodeType: string;
  input: any;                 // 入参 JSON
  prompt?: string;            // 实际发送给大模型的 Prompt
  rawResponse?: string;       // 大模型原始返回
  structuredOutput?: any;     // 处理后结构化输出
  startTime: number;
  endTime: number;
  durationMs: number;
  status: 'pending' | 'running' | 'success' | 'failed';
  error?: string;             // 节点级错误信息
}
 
实体3：Prompt模板 Template

{
  templateId: string;
  sceneTag: string[]; // 场景标签，用于自动匹配
  content: string; // 预设模板文本，预留任务填充占位符
  outputSchema: object; // 强制输出JSON格式约束
}

八、技术选型与设计决策细化（已确认）

8.1 技术栈选型

| 层面 | 选型 | 说明 |
| --- | --- | --- |
| 前端框架 | React 18+ | 配合 React Router 实现 SPA 路由 |
| 可视化画布 | AntV X6 | 蚂蚁开源免费，基于 SVG/HTML，React 集成友好 |
| 构建工具 | Vite | 快速开发服务器，ESBuild 预打包 |
| 后端框架 | FastAPI (Python) | 轻量异步，提供大模型 API 转发服务 |
| 大模型 | DeepSeek | 保留多模型切换扩展能力 |
| UI 组件库 | 待定 | 建议 Ant Design 或 shadcn/ui，与 X6 风格统一 |

8.2 API Key 管理方案

- 后端提供 `POST /api/config/key` 接口，用户首次使用时填入自己的 API Key
- 后端将 Key 加密后存入临时文件（如 `/tmp/masm_key.enc`）
- 重启服务后临时文件丢失，用户需重新输入
- 前端不存储任何 Key 信息，所有 LLM 调用经后端转发
- 优点：MVP 足够简单且安全，Key 不暴露在浏览器端

8.3 节点拓扑连接约束

校验规则：保存或运行前，检查每个节点是否满足下表约束，不满足则给出明确提示。

| 节点类型 | 入边数量 | 出边数量 | 说明 |
| --- | --- | --- | --- |
| 开始节点 | 0 | 1 | 唯一入口，全局仅允许一个 |
| 结束节点 | ≥1 | 0 | 可多个节点指向它 |
| Agent 节点 | ≥1 | 1 | MVP 限制为单出 |
| 条件分支节点 | ≥1 | 2 | 固定两路：通过/不通过 |
| 并行分支节点 | ≥1 | ≥2 | 至少两条子链路，最大可配置（默认10） |
| 汇总节点 | ≥2 | 1 | 至少两个入边，合并后单出 |

悬挂节点策略：
- 画布编辑阶段：允许拖拽节点到画布但不连线，作为草稿
- 保存/运行前校验：所有节点必须处于从开始到结束的连通路径中，否则提示"存在未连接的节点，请删除或连接"

8.4 并行执行与汇总策略

- 并行执行：使用 `Promise.all` 同时发起所有分支的 LLM 调用，实现真并发
- 汇总等待：汇总节点通过 `await Promise.all(branchPromises)` 自然等待所有分支完成
- 超时兜底：每个分支执行时携带剩余总时间的计时，若某分支超时则整体失败并记录日志
- 数据合并：MVP 阶段汇总节点做纯 JSON 拼接（将所有分支输出收集为数组），不额外调用 AI 合并。后续版本可加入 AI 合并开关

8.5 页面路由与导航设计（SPA）

四个核心页面：

| 路由 | 页面 | 功能 |
| --- | --- | --- |
| `/` | 蓝图列表页 | 展示所有已保存蓝图，支持新建、删除、导入/导出 |
| `/editor/:id` | 画布编辑页 | 核心拖拽编辑器，左侧节点库+右侧配置抽屉，主Agent智能生成按钮 |
| `/execute/:id` | 流程执行页 | 独立运行已保存蓝图，实时高亮节点 |
| `/logs/:logId` | 日志详情页 | 展示单次执行完整日志与节点级记录 |
| `/settings` | API 设置页 | LLM 供应商选择、API Key 配置、状态检测与清除 |

导航关系：
```
列表页 → 点击"新建/编辑" → 编辑页 → 点击"运行" → 执行页 → 执行完成 → 日志详情页
各页面均有返回按钮回到上一级
```

8.6 节点配置交互方式

- 采用右侧抽屉（Drawer）：点击画布节点后，右侧滑出配置面板
- 优点：不遮挡画布主体，空间充足可容纳表单、文本域等
- 交互：抽屉内含"保存"和"取消"按钮，支持失焦自动保存

8.7 撤销操作规格

- 撤销范围：节点增删、连线增删、节点移动
- 文本修改粒度：以"节点配置面板失去焦点（blur）"或"点击保存"为一次操作，非逐字符记录
- 撤销栈上限：最近 50 步（可配置）
- 无重做（Redo），仅支持撤销（Undo）
